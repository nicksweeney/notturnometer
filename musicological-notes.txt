Musicological notes — close calls in work identity
===================================================

A log of the judgment calls behind work-grouping decisions that the
canonicalization rules can't fully automate: the cases where two recordings or
titles are arguably the same work and arguably distinct, and a human had to
decide. Each entry records the case, the evidence (favour the performance-PID
fingerprint over title strings), the decision, the reasoning, and any rule it
sets or tests.

PROCESS: musicological close calls get an adversarial review from "Cerys" (a
harp-scholar persona; spawn the `cerys` agent) BEFORE the decision is finalised.
Pose the PURELY musicological hypotheses, stripped of database framing; demand
pushback. Record their verdict in the entry. (Cerys overturned the instrument-
family rule below and supplied the transcription rubric.)


-------------------------------------------------------------------------------
Transcription depth as the work boundary  (violin/cello vs keyboard variants)
2026-06-15   [supersedes the instrument-family rule, overturned same day]
-------------------------------------------------------------------------------

Case. Franck, Sonata in A major (M.8 / FWV.8). Composed for violin (1886);
Delsart's cello transcription (1888, Franck-sanctioned) is the standard cello
version. In the corpus it fragments 11 ways / 226 airings, on three axes:
catalogue (M.8 / FWV.8 / none), spelling, and instrument (violin / cello /
"violin or cello" / "either violin or cello").

Evidence (performance PIDs). 18 distinct recordings: 11 violin, 7 cello. The
performer credit is ALWAYS unambiguous — a cellist plays cello even under a
"violin or cello" title — whereas the segment title is not reliable for the
instrument. Durations overlap completely (violin 1530-1815s, cello 1672-1787s):
one ~27-minute work. So the instrument can only be recovered from the performer
fingerprint, never from the title.

First call (Nick, OVERTURNED). Instrument FAMILY is the boundary: keyboard
(organ/piano/harpsichord/fortepiano) interchangeable for one player -> MERGE;
non-keyboard (violin/cello/winds) -> SPLIT, since a cellist doesn't play violin.
M.8 would split violin/cello, deferred to the fingerprint oracle.

OVERTURNED (Cerys adversarial consult, 2026-06-15). The instrument-family line
"isn't tracking anything musical": harpsichord->piano changes voicing/dynamics
MORE than violin->cello; it would fragment the catalogue (the Arpeggione sonata,
Brahms's own viola versions of the clarinet sonatas, the Bach cello suites all
"different works"); and the harp breaks it (harp transcriptions of Debussy piano
works are neither keyboard nor orchestral string/wind). The real axis is DEGREE
OF RECOMPOSITION, independent of instrument:

  POLICY (adopted). Work identity = transcription depth, three tiers:
    - LITERAL transcription (same notes, new instrument) -> SAME work, version tag.
      (Delsart cello Franck; Brahms's own viola versions; Bach suites across the
      string family; Haydn keyboard-vs-piano trios.)
    - Composer's OWN re-scoring for new forces -> same work, FLAGGED. (Brahms
      2-piano vs orchestral Haydn Variations; Bach BWV1058/1042; Ravel's Pictures
      is a third-party orchestration = still same work, orchestrator is a credit.)
    - FREE PARAPHRASE / fantasy -> a NEW, distinct work. (Liszt operatic
      paraphrases, Busoni-Bach.) Compositional intervention is the boundary.

Consequences. The keyboard merges already shipped (54937e7, Haydn/Scarlatti) STAND
— "same notes, different keyboard" is a literal transcription = same work under
the new rule too; no reversal. M.8 (Franck Sonata in A) cello version MERGES with
the violin original (straight Delsart transcription) — the violin/cello split is
DROPPED, the oracle is no longer needed for it. M.8 untangle SHIPPED 2026-06-15
(ed48b9a): 10 aliases fold all 11 keys onto "Violin Sonata in A major, M.8",
one 226x work; the cello version is a display facet.

OPERATIONAL RUBRIC (Cerys, for a system seeing only title + performer + duration).
Key insight — the "advertising asymmetry": deep recompositions ADVERTISE
themselves (the marker is the whole selling point of a paraphrase), while shallow
transcriptions go unmarked. So DEFAULT TO MERGE; split-markers are the active
ingredient. "Look for a reason to split; absent a marker, merge." Decision order:
SPLIT -> ESCALATE -> else MERGE.

  SPLIT (new distinct work):
    - "paraphrase" / "concert paraphrase" / "réminiscences de" — gold.
    - "fantasy/fantasia/fantaisie/Fantasie" + "on/über/sur" + a named WORK — the
      PREPOSITION carries it; bare "Fantasia" is just a form-title -> merge.
    - "concert transcription / transcription de concert" — the "concert" qualifier
      signals showpiece recomposition.
    - "freely after / frei nach / librement" — the ADVERB "freely" is the trigger
      (bare "after"/"based on"/"d'après"/"nach" is overloaded -> ESCALATE).
    - "on themes from <named work>" + a virtuoso solo instrument in the credits.
  MERGE (same work, version is a display facet — the DEFAULT):
    - NO recomposition marker -> merge under the stated composer.
    - "arr." / "transcr." / "orch." / "version for" / "for <instruments>" /
      "realised" / "ed./edition" (and NOT "concert"/"freely").
    - Composer's own re-scoring -> merge, flag. Different duration/instruments are
      DISPLAY FACETS, never a work split.
    - *** BORROWED-THEME GUARD (drill this in): the COMPOSER FIELD is the author;
      a theme-source named in the title is NEVER the author and never the filing
      home. "variations/rhapsody on a theme OF/BY <a PERSON>" (Brahms Haydn/
      Paganini Variations, Rachmaninov Paganini Rhapsody, Liszt Totentanz) =
      ORIGINAL work -> merge under the credited composer, NEVER file as an excerpt
      of the theme-source. Tell: prepositional object is a PERSON -> borrowed theme
      (merge); object is a WORK/opera -> paraphrase (consider split).
  ESCALATE (title can't see the depth): bare "after/based on" w/o "freely";
    "transcription" on a known Liszt/Busoni item; one piece under clashing EBU
    house styles; cadenzas/completions/editions (Mozart Requiem Süssmayr/Levin,
    Mahler 10, Art of Fugue); famous-composer surname used adjectivally with no
    "of". -> flag for a human, don't guess.

Downstream note (still relevant). `_strip_arrangement_tail` folding "arr./transc.
for X" -> original (371 titles / 1103 airings) is CONSISTENT with the new policy
(arrangements are same-work). No change needed there.
